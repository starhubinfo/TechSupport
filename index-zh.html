FlClash 技术支持

基于 mihomo 内核的 iOS 网络配置管理工具。查看功能说明、系统要求与常见问题排查。

版本 1.0.0·需要 iOS 15.0 或更高版本·支持 简体中文 / English

主要功能

FlClash 把「配置管理」与「流量接管」分开：不开启 VPN 也能加载配置、查看策略组、切换节点。

实时仪表盘

实时显示上行 / 下行速率、连接数、出站模式与进程内存占用，并可一键重启内核。

策略组与节点

查看全部策略组，手动切换节点，实时测试各节点延迟，支持规则 / 全局 / 直连模式切换。

配置订阅管理

支持 URL 订阅下载、本地文件导入、二维码扫描添加；多份配置自由切换，热应用无需重启。

按需开启 VPN

仅在需要真正代理流量时开启系统 VPN（Network Extension），其余时间仅作本地配置管理。

连接与日志

查看活动连接明细与内核实时日志，支持日志级别调整、网络诊断与一键清理。

Geo 数据与高级设置

GeoIP / GeoSite 数据自动更新，可调整混合端口、IPv6、TCP 并发、Hosts、低内存加载器等。

常见问题

大部分使用问题都可以在这里找到答案。

如何导入我的配置文件？

进入「配置」页，点击右上角 +，可选择三种方式：

  1. 从 URL 下载订阅 — 粘贴订阅链接后点击下载。
  2. 从文件导入 — 选择设备上已保存的 .yaml / .yml 文件。
  3. 扫描二维码 — 使用相机扫描配置二维码。

导入后点击配置项即可启用，配置会热应用，无需重启应用。

开启 VPN 时提示「需要允许 VPN 配置」怎么办？

首次开启时，iOS 会弹出系统授权对话框，需要点击「允许」并通过 Face ID / Touch ID / 密码验证。这是 iOS 的标准流程，用于将 VPN 配置写入系统设置。

若误点了「不允许」，请在 iOS 设置 → 通用 → VPN 与设备管理 中删除残留配置后，回到应用重新开启。

VPN 已开启，但网络无法访问

请按顺序排查：

  1. 在「仪表盘」确认状态显示为「正在接管流量」。
  2. 在「代理」页对当前策略组测速，确认所选节点延迟正常（非 Timeout）。
  3. 确认配置文件未过期，可在「配置」页下拉刷新订阅。
  4. 切换出站模式（规则 / 全局）验证是否为分流规则问题。
  5. 在「工具 → 日志」将级别调至 info 或 debug，查看具体报错。
  6. 仍无效时，关闭 VPN 后在「工具 → VPN 诊断」中运行一次检测。
使用一段时间后 VPN 自动断开

iOS 对网络扩展进程有约 50MB 的内存硬上限，超出会被系统终止。应用已内置多层内存控制，但在超高并发场景下仍可能触发。建议：

  • 在「工具 → 基本配置」中开启低内存 Geo 加载器（memconservative）。
  • 将日志级别设为 warning 或 silent。
  • 关闭 TCP 并发与不必要的 IPv6。
  • 精简配置中的规则集，优先使用 GeoIP 而非大体积 GEOSITE 规则集。

可在「仪表盘」实时查看扩展进程的内存占用。

如何切换应用语言？

进入「工具 → 语言」，可选择跟随系统、中文或 English。部分底部标签栏文字需要重启应用后完全生效。

Geo 数据（GeoIP / GeoSite）如何更新？

进入「工具 → 资源」，可查看各数据文件的下载状态与更新时间，点击即可手动更新。开启自动更新后，应用会按设定的间隔（小时）在后台刷新。

应用会收集我的数据吗？

不会。FlClash 不收集、不上传、不分析任何个人信息或使用数据，也不含第三方统计与广告 SDK。您添加的订阅配置、连接日志与流量统计仅存储在设备本地，可随时在应用内清除。

详情请查看隐私政策。

如何彻底卸载并清除数据？
  1. 在应用内关闭 VPN 开关。
  2. 长按主屏幕图标 → 删除 App。
  3. 前往 iOS 设置 → 通用 → VPN 与设备管理，删除名为 FlClash 的 VPN 配置。

删除应用后，所有本地配置与日志将随沙盒一并移除。

提交问题反馈时应提供哪些信息？

为便于快速定位，请在邮件中附上：

  • 设备型号与 iOS 版本（如 iPhone 14、iOS 17.5）
  • 应用版本号（可在「工具 → 关于」中查看）
  • 问题的具体表现与复现步骤
  • 「工具 → 日志」中的相关日志片段（请自行隐去订阅链接、服务器地址等敏感信息）
